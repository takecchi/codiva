# 多言語対応（i18n）規約

UI に見えるすべての文字列は言語カタログに集約し、設定で日本語/英語を切り替えられる状態を保つ。
新しい画面・メッセージを足すときは、必ず両言語ぶんをカタログに追加する。

## 大原則

- **UI コンポーネント（`.tsx`）に表示文字列を直書きしない。** 文字列は `src/core/i18n.ts` の
  `messages` カタログ（`Messages` 型）にのみ置く。UI からは `useMessages()` で引く。
  - 例外は「表示テキストではない」もの: 色名（`'cyan'`）、記号（`'❯'`, `'» '`, `'y'`/`'n'` などのキー）、
    数値・ID・ブランチ名等のデータ。これらは翻訳対象ではない。
- **ja と en は必ず対で追加する。** 片方だけ足さない。`Messages` 型が欠落を型エラーで捕え、
  `i18n.spec.ts` が ja/en のキー集合一致を実行時にも検証する（新グループを足してもこのテストが番人になる）。
- **カタログは純粋に保つ。** `core/i18n.ts` は Ink/React/fs/execa を import しない（[architecture.md](./architecture.md)）。

## レイヤ配置

| 対象 | 置き場所 |
|------|---------|
| 文字列カタログ・`Lang`・`resolveLang`・`detectLocaleLang`・`normalizeLang` | `src/core/i18n.ts`（純粋） |
| 設定ドメイン型・JSON 検証（`CodivaConfig` / `toConfig`） | `src/core/config.ts`（純粋） |
| 設定ファイルの読み書き（`~/.codiva/config.json`） | `src/utils/config.ts`（I/O ラッパ） |
| React コンテキスト（`MessagesProvider` / `useMessages`） | `src/ui/i18n-context.tsx` |

## 文字列を追加・変更する手順

1. `Messages` interface に適切なグループ（`list` / `detail` / `badge` / `permission` / `app`）でキーを足す。
2. `ja` と `en` **両カタログ**に実装を書く。
3. 動的な差し込み・複数形は**文字列テンプレート関数**にする（例: `(n: number) => string`）。
   これで型安全（`any` 不要）・言語ごとの語順や複数形差を吸収できる。UI 側で `${}` 連結して組み立てない。
4. UI から参照する:
   - コンポーネントは `const m = useMessages()` → `m.detail.notFound` のように使う。
     `m` はメッセージカタログ用に予約（`s`=session, `t`=todo などと衝突させない）。
   - 純関数（`badgeFor` 等）は `Messages` を**引数で受ける**（フックを呼ばない）。
5. 色付きキー（`y`/`n`/`m`/`d`）を色付き `<Text>` で挟む行は、**キー記号は JSX のまま、語だけカタログ化**する。
6. 文字列を変えたら、対応する spec / `tests/*.test.tsx` のアサートも更新する。

## 言語の決定（優先順位）

`resolveLang()` が次の順で決める（上が優先）:

1. 環境変数 `CODIVA_LANG`（`ja` / `en`。明示上書き）
2. 設定ファイル `~/.codiva/config.json` の `language`（`'ja'` / `'en'`。`'auto'` は次へ委譲）
3. OS ロケール（`LC_ALL` → `LC_MESSAGES` → `LANG`）: `ja*` なら日本語、それ以外は英語

配線は `src/index.tsx`（合成ルート）で行い、解決済みカタログを `App` の `messages` prop に注入する。

## 言語を増やすとき

`core/i18n.ts` の `Lang` 型・`LANGS`・`messages` に言語を追加し、`normalizeLang` /
`detectLocaleLang` の判定を拡張する。UI 側は `useMessages()` 経由なので変更不要。

## 落とし穴

- ブロックコメント内に `*/` を含む文字列（例: `ja*` と `/en` が隣接）を書くとコメントが早期終了する。
  oxc（vitest の変換器）が `Invalid Character` で落ちる。語の間にスペースを入れて回避する。
